Title: Flipped Classroom Using Retrieval-Augmented Generation and Machine Learning for M.Tech CSE Education

Authors: Uttam Vitthal Bhise
Affiliation: M.Tech, Computer Science and Engineering
Academic Year: 2025-2026

Abstract
This paper presents FlipLearn, a full-stack learning platform that combines the flipped classroom model with Retrieval-Augmented Generation (RAG) and machine learning (ML) for postgraduate computer science education. In a flipped classroom, students study learning material before class and engage in problem-solving during class. FlipLearn operationalizes this model through video lectures, study materials, quizzes, assignments, analytics dashboards, and an AI tutor. The RAG pipeline retrieves domain-grounded context from curated knowledge notes, textbooks, uploaded PDFs, and quiz content before generating responses with a large language model. In parallel, ML models estimate student outcomes and identify at-risk learners using engagement and assessment features. The system is implemented in Django with role-based workflows for students, teachers, and administrators. Evaluation focuses on platform functionality, model behavior, and practical educational impact. Results indicate that integrating RAG with analytics-enabled flipped pedagogy improves learner support, enables early intervention, and strengthens data-informed teaching.

Keywords: Flipped Classroom, Retrieval-Augmented Generation, Learning Analytics, Educational Data Mining, Student Performance Prediction, At-Risk Detection

1. Introduction
Traditional lecture-centric teaching often limits active learning time and individualized support. In contrast, the flipped classroom shifts primary content delivery outside class and reserves contact hours for discussion, exercises, and higher-order learning. While this model improves engagement, its effectiveness depends on timely learner support, resource accessibility, and continuous performance monitoring.

Recent advances in AI and analytics make it feasible to build intelligent educational systems that can:
1. Provide contextual tutoring support.
2. Track student engagement signals.
3. Predict learning outcomes and risk.
4. Assist teachers through actionable dashboards.

This work introduces FlipLearn, an integrated platform for M.Tech CSE learners that combines:
- Flipped classroom content workflows.
- A subject-grounded RAG chatbot.
- ML-based performance prediction and at-risk alerts.

The key objective is to improve both instructional quality and student outcomes through a single deployable system.

2. Problem Statement
Many institutions adopting flipped learning face three recurring challenges:
1. Students cannot always get immediate conceptual help while studying alone.
2. Teachers lack unified analytics to identify weak learners early.
3. Educational content, assessment, and prediction tools are often fragmented across separate systems.

This project addresses these challenges by developing one integrated web platform that supports content delivery, assessment, AI tutoring, and predictive analytics.

3. Objectives
The main objectives of this research are:
1. Design and implement a role-based flipped classroom platform.
2. Build a RAG chatbot grounded in course-specific sources.
3. Train and use ML models for score prediction and risk classification.
4. Provide actionable dashboards for teachers and students.
5. Evaluate practical utility in a postgraduate CSE context.

4. Related Work Summary
Research in technology-enhanced education shows that flipped classrooms improve participation and conceptual retention when combined with active tasks. Educational data mining studies report that engagement features such as attendance, quiz performance, and submission patterns are useful predictors of academic outcomes. Recent RAG approaches improve factuality by retrieving external context before generation, reducing purely parametric hallucination in LLM answers.

However, many prior systems handle these capabilities separately. This work contributes a unified pipeline where flipped content workflows, grounded tutoring, and predictive analytics coexist in one application.

5. System Architecture
FlipLearn follows a modular architecture:

5.1 Application Layer
- Backend: Django (Python)
- Frontend: HTML, CSS, Bootstrap, JavaScript
- Authentication: Django auth with role-aware dashboards

5.2 Data Layer
- Development DB: SQLite
- Production DB: PostgreSQL (via environment-based configuration)
- Media storage: Local or Cloudinary-based depending on deployment settings

5.3 AI/ML Layer
- RAG index construction: FAISS + sentence-transformers embeddings
- Generation model access: Groq API
- ML models: Random Forest Regressor/Classifier and comparative baselines

5.4 User Roles
1. Student
   - Access videos/materials
   - Attempt quizzes and submit assignments
   - Use AI tutor and view personal performance
2. Teacher
   - Upload content
   - Create quizzes/assignments
   - Grade and monitor class analytics
3. Admin
   - Access global dashboard
   - Manage system-level operations

6. Methodology
6.1 Flipped Learning Workflow
1. Students consume video/material resources before class.
2. Students complete quizzes and assignments.
3. During class, teachers focus on discussion, clarification, and application tasks.
4. The platform continuously captures engagement and performance records.

6.2 RAG Pipeline
1. Source ingestion from:
   - Subject notes (text files)
   - Textbook PDFs
   - Uploaded study materials
   - Quiz question data
2. Text chunking with overlap.
3. Embedding creation and FAISS index storage.
4. Query-time retrieval of top-k relevant chunks.
5. Prompt assembly with retrieved context and optional chat history.
6. LLM response generation and streaming to client.

6.3 ML Prediction Pipeline
Input features include:
- Videos watched
- Total video watch time
- Quiz average score
- Assignment average marks
- Attendance percentage
- Participation score (including chat interactions)
- Previous GPA

Outputs include:
- Predicted score (regression)
- Predicted performance label (classification)
- At-risk flag
- Confidence estimate

6.4 Real-Time Update Mechanism
Django signals and view-level updates refresh student performance records after key events such as quiz attempt, assignment grading, and video completion.

7. Implementation Details
7.1 Core Modules
- flipped_app: web logic (views, models, forms, templates)
- rag_engine: indexing, retrieval, and chat orchestration
- ml_model: dataset generation, training, and prediction utilities

7.2 Security and Reliability Practices
- Environment-variable-driven secrets and host configuration
- Role-based authorization checks on sensitive endpoints
- Safer error handling in user-facing APIs
- Regression tests for access control and due-date enforcement
- Static file collection sanity fixes to avoid duplicate path collisions

7.3 Deployment
The platform supports cloud deployment (Render/Railway patterns) with migration and static collection steps configured for production startup.

8. Experimental Observations
8.1 Functional Evaluation
The system successfully supports end-to-end flipped classroom operations:
- Content publishing and consumption
- Assessments and grading
- AI-assisted conceptual support
- Analytics and risk alerts

8.2 RAG Behavior
Grounded retrieval improves contextual relevance of responses, especially for domain-specific CSE topics. Response quality is strongest when source coverage is adequate and chunking is balanced.

8.3 ML Behavior
Random Forest-based models provide practical prediction performance with interpretable feature relationships. Engagement-related features (quiz score, attendance, and assignment marks) show strong influence on risk outcomes.

9. Discussion
The integrated architecture demonstrates that flipped pedagogy benefits from both conversational support and predictive analytics. Teachers gain early-warning insight, and students get immediate help while studying independently. The unified platform also reduces operational overhead compared to disconnected tools.

Limitations include:
- Dataset scale constraints in early-stage deployment.
- Dependence on quality and freshness of indexed learning content.
- Need for periodic model retraining and calibration across cohorts.

10. Conclusion
This research presents a practical, deployable framework for AI-enabled flipped learning in postgraduate CSE education. By combining RAG-based tutoring and ML-driven analytics in a single platform, FlipLearn improves learner support, enables earlier interventions, and helps instructors make data-informed decisions. The framework is extensible to other disciplines and institutions with moderate adaptation.
